# RAG 智能知识库问答系统 — 项目介绍

**版本**: v2.0.0 | **最后更新**: 2026-06-30 | **Java**: 21 | **Spring Boot**: 3.5.0 | **Spring AI**: 1.1.7 | **AgentScope**: 2.0.0-RC3

---

## 项目概述

基于 Spring AI + AgentScope HarnessAgent 构建的企业级 RAG（检索增强生成）知识库系统，支持**双引擎并行**：Workflow RAG（固定管道）和 **Agentic RAG**（自主推理 Agent），以及记忆管理、质量评估和全方位可观测性。

---

## 核心功能

### RAG 智能问答

| 功能 | 说明 |
|------|------|
| **Workflow RAG** | 11 阶段固定管道（清洗->增强->多查询->并行检索->CRAG->去重->过滤->重排序->压缩->生成）|
| **Agentic RAG** | HarnessAgent 自主 ReAct 循环 + 查询分解 + Step-Back 查询 + 多工具调用 |
| **质量保障管道** | SufficientContextAgent -> SelfReflection -> CorrectiveRepair -> LLM Judge |
| **查询路由** | QueryRouter：规则(长度+关键词) + LLM 两级分类，决策 Workflow vs Agentic |
| **混合检索** | 向量相似度（Milvus）+ BM25 关键词（本地索引），RRF 算法融合 |
| **多阶段管道** | 11 个独立阶段，按序执行，支持条件跳过 |
| **上下文压缩** | Embedding 粗压缩 + LLM 精压缩两级策略 |

### Agentic RAG 特性

| 特性 | 说明 |
|------|------|
| **HarnessAgent 引擎** | AgentScope 自主推理 ReAct 循环，内置工作区、工具、会话持久化 |
| **工具框架** | 5 个 AgentTool：vector_search、bm25_search、external_search、sql_query、memory_query |
| **查询分解** | TransformationGate + QueryDecomposer，复合问题拆解为原子子查询 |
| **Step-Back 查询** | 检索结果不足时自动生成更宽泛的查询 |
| **中间件管道** | ToolCallCaptureMiddleware 拦截工具调用，记录到轨迹 |
| **轨迹记录** | 6 阶段 StepRecord + 轨迹持久化到 MySQL |
| **SSE 流式输出** | GET/POST 双模式，实时推送 thinking/token/check/done 事件 |

### 记忆管理

| 类型 | 存储 | 特性 |
|------|------|------|
| **短期记忆** | Redis + 内存二级缓存 | 30 分钟 TTL，最多 20 条/会话 |
| **长期记忆** | MySQL + 向量检索 | LLM 提取（事实/偏好/上下文），余弦相似度合并 |

### 企业级安全

- JWT 双 Token 机制（Access Token 24h + Refresh Token 7天）
- BCrypt 密码加密
- Spring Security + CORS 配置
- ASYNC 派发自动排除（避免 SSE 清理时报 AccessDenied）

## 系统架构

### 三层架构

```
┌──────────────────────────────────────────────┐
│           Presentation Layer                  │
│  AuthController / KnowledgeQAController       │
│  GlobalExceptionHandler / RateLimitFilter     │
├──────────────────────────────────────────────┤
│           Business Logic Layer                │
│  RagFlow → RagPipeline → PipelineStage × 11  │
│  RagOrchestrator / RetrievalStrategy          │
│  MemoryManagers / EvaluationManager           │
├──────────────────────────────────────────────┤
│           Data Access Layer                   │
│  JPA Repository / VectorStore / RedisTemplate │
├──────────────────────────────────────────────┤
│           Infrastructure Layer                │
│  MySQL / Redis / Milvus / DashScope API       │
└──────────────────────────────────────────────┘
```

### RAG 执行流程

```
用户请求 → JWT 认证
         → RagOrchestrator.beforeExecute()
             缓存检查 (命中则直接返回)
             加载短期/长期记忆
             分类问题复杂度 (SIMPLE/MODERATE/COMPLEX)
         → RagPipeline.execute()
             ① QueryCleaningStage        — 去空格、规范化空白
             ② QueryEnhancementStage     — 记忆指代消解 + 关键词扩展
             ③ MultiQueryGenerationStage — 多角度查询生成 (仅复杂问题)
             ④ ParallelRetrievalStage    — 并行多路向量检索
             ⑤ RetrievalStage            — 单次向量检索 (兜底)
             ⑥ CragStage                 — 检索质量评估与修正
             ⑦ DeduplicationStage        — 哈希去重 + Jaccard 模糊去重
             ⑧ FilteringStage            — 移除过短文挡 (<20字符)
             ⑨ ReRankingStage            — MMR 重排序
             ⑩ CompressionStage          — Embedding/LLM 两级压缩
             ⑪ GenerationStage           — LLM 生成答案 + 引用提取
         → RagOrchestrator.afterExecute()
             保存短期记忆 → 提取长期记忆 → 异步评估 → 缓存结果
         → 返回 RagAnswer (答案 + 来源列表)
```

### 两种流程实现

| 特性 | BasicRagFlow | AdvancedRagFlow | AgenticRagFlow |
|------|-------------|----------------|
| **管道阶段数** | 7 个 | 11 个 | Agent 自主决策 |
| **短期记忆** | ❌ 禁用 | ✅ 启用 | HarnessAgent内置 |
| **长期记忆** | ❌ 禁用 | ✅ 启用 |
| **质量评估** | ❌ 禁用 | ✅ 启用 |
| **关键词扩展** | ❌ | ✅ |
| **多查询生成** | ❌ | ✅ |
| **并行检索** | ❌ | ✅ |
| **CRAG** | ❌ | ✅ |
| **MMR 重排序** | ❌ | ✅ |

| 包 | 组件 | 说明 |

| `rag/orchestrator/` | `DefaultRagOrchestrator` | 横切关注点协调 |
| `memory/` | `ShortTermMemoryManager` | 短期记忆（Redis + 内存） |
| `memory/` | `LongTermMemoryManager` | 长期记忆（MySQL + 向量） |
| `evaluation/` | `RagEvaluator` | LLM 驱动的 RAGAS 评估 |
