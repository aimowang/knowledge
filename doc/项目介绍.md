# RAG 智能知识库问答系统 — 项目介绍

**版本**: v1.0.0 | **最后更新**: 2026-06-14 | **Java**: 21 | **Spring Boot**: 3.5.0 | **Spring AI**: 1.1.7

---

## 项目概述

基于 Spring AI 构建的企业级 RAG（检索增强生成）知识库系统，支持智能问答、记忆管理、质量评估和全方位可观测性。采用 **管道模式 + 策略模式 + 编排器模式**，将核心流程从 910 行简化为 98 行。

---

## 核心功能

### RAG 智能问答

| 功能 | 说明 |
|------|------|
| **混合检索** | 向量相似度（Milvus）+ BM25 关键词（本地索引），RRF 算法融合 |
| **多阶段管道** | 11 个独立阶段，按序执行，支持条件跳过 |
| **纠错机制** | CRAG：LLM 评估检索质量，模糊时精炼重查，错误时外部搜索 |
| **MMR 重排序** | 最大边际相关性算法，平衡相关性与多样性 |
| **上下文压缩** | Embedding 粗压缩 + LLM 精压缩两级策略 |
| **多查询生成** | 复杂问题自动生成多角度查询，并行检索 |
| **关键词扩展** | 同义词扩展提升召回率 |

### 记忆管理

| 类型 | 存储 | 特性 |
|------|------|------|
| **短期记忆** | Redis + 内存二级缓存 | 30 分钟 TTL，最多 20 条/会话 |
| **长期记忆** | MySQL + 向量检索 | LLM 提取（事实/偏好/上下文），余弦相似度合并 |

### 质量评估

- **5 项指标**：答案相关性、忠实度、上下文相关性、上下文精确率、上下文召回率
- **异步评估**：不阻塞主流程，通过编排器开关控制
- **定期分析**：每天凌晨 3 点自动触发低质量模式分析

### 企业级安全

- JWT 双 Token 机制（Access Token 24h + Refresh Token 7天）
- BCrypt 密码加密
- API 速率限制（Bucket4j，默认 100 req/min/IP）
- Spring Security + CORS 配置

## 系统架构

### 三层架构

```
┌──────────────────────────────────────────────┐
│           Presentation Layer                  │
│  AuthController / KnowledgeQAController       │
│  GlobalExceptionHandler / RateLimitFilter     │
├──────────────────────────────────────────────┤
│           Business Logic Layer                │
│  RagFlow → RagPipeline → PipelineStage × 11  │
│  RagOrchestrator / RetrievalStrategy          │
│  MemoryManagers / EvaluationManager           │
├──────────────────────────────────────────────┤
│           Data Access Layer                   │
│  JPA Repository / VectorStore / RedisTemplate │
├──────────────────────────────────────────────┤
│           Infrastructure Layer                │
│  MySQL / Redis / Milvus / DashScope API       │
└──────────────────────────────────────────────┘
```

### RAG 执行流程

```
用户请求 → JWT 认证
         → RagOrchestrator.beforeExecute()
             缓存检查 (命中则直接返回)
             加载短期/长期记忆
             分类问题复杂度 (SIMPLE/MODERATE/COMPLEX)
         → RagPipeline.execute()
             ① QueryCleaningStage        — 去空格、规范化空白
             ② QueryEnhancementStage     — 记忆指代消解 + 关键词扩展
             ③ MultiQueryGenerationStage — 多角度查询生成 (仅复杂问题)
             ④ ParallelRetrievalStage    — 并行多路向量检索
             ⑤ RetrievalStage            — 单次向量检索 (兜底)
             ⑥ CragStage                 — 检索质量评估与修正
             ⑦ DeduplicationStage        — 哈希去重 + Jaccard 模糊去重
             ⑧ FilteringStage            — 移除过短文挡 (<20字符)
             ⑨ ReRankingStage            — MMR 重排序
             ⑩ CompressionStage          — Embedding/LLM 两级压缩
             ⑪ GenerationStage           — LLM 生成答案 + 引用提取
         → RagOrchestrator.afterExecute()
             保存短期记忆 → 提取长期记忆 → 异步评估 → 缓存结果
         → 返回 RagAnswer (答案 + 来源列表)
```

### 两种流程实现

| 特性 | BasicRagFlow | AdvancedRagFlow |
|------|-------------|----------------|
| **管道阶段数** | 7 个 | 11 个 |
| **短期记忆** | ❌ 禁用 | ✅ 启用 |
| **长期记忆** | ❌ 禁用 | ✅ 启用 |
| **质量评估** | ❌ 禁用 | ✅ 启用 |
| **关键词扩展** | ❌ | ✅ |
| **多查询生成** | ❌ | ✅ |
| **并行检索** | ❌ | ✅ |
| **CRAG** | ❌ | ✅ |
| **MMR 重排序** | ❌ | ✅ |

## 模块说明

| 模块 | 路径 | 职责 |
|------|------|------|
| **api** | `api/` | REST 控制器、全局异常处理 |
| **core** | `core/` | 核心业务逻辑、RAG 流程、记忆、评估、检索 |

| **starter** | `starter/` | Spring Boot 启动类、配置、DB schema |

### 关键组件一览

| 包 | 组件 | 说明 |
|----|------|------|
| `retrieval/` | `HybirdContentRetriever` | 混合检索器（@Primary），向量+BM25 RRF 融合 |
| `retrieval/` | `Bm25Indexer` | 轻量级 BM25 搜索引擎，支持中英文 |
| `retrieval/` | `BasicContentRetriever` | 纯向量检索 |
| `retrieval/` | `RetrievalOptimizer` | LLM 查询扩展/重写/关键词提取 |
| `compress/` | `HybridCompressor` | 两级压缩（Embedding + LLM） |
| `rag/pipeline/stage/` | 11 个 PipelineStage | 每个阶段单一职责 |
| `rag/strategy/` | 3 个策略接口 + 6 个实现 | 可插拔策略 |
| `rag/orchestrator/` | `DefaultRagOrchestrator` | 横切关注点协调 |
| `memory/` | `ShortTermMemoryManager` | 短期记忆（Redis + 内存） |
| `memory/` | `LongTermMemoryManager` | 长期记忆（MySQL + 向量） |
| `evaluation/` | `RagEvaluator` | LLM 驱动的 RAGAS 评估 |
