```bash

TAVILY_API_KEY=                             # Tavily Search API Key
BING_API_KEY=                               # Bing Search API Key
```

## 快速启动

### 1. 克隆与配置
```bash
git clone <repository-url>
cd knowledge
cp .env.example .env
# 编辑 .env 填入必要的密钥
```

### 2. 启动基础设施
```bash
# 启动核心服务（MySQL, Redis, Prometheus, Grafana, Zipkin）
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 3. 编译应用
```bash
mvn clean package -DskipTests
# JAR 路径: starter/target/starter-1.0-SNAPSHOT.jar
```

### 4. 启动应用

**开发环境** (基础设施在 localhost):
```bash
java -jar starter/target/starter-1.0-SNAPSHOT.jar --spring.profiles.active=dev
```

**生产环境** (基础设施在 Docker 网络):
```bash
java -jar starter/target/starter-1.0-SNAPSHOT.jar --spring.profiles.active=prod
```

### 5. 验证
```bash
# 健康检查
curl http://localhost:8080/actuator/health

# 预期响应: {"status":"UP"}
```

## 外部 API

### DashScope (阿里云通义千问)

本系统依赖阿里云 DashScope 提供 LLM / Embedding / Rerank 服务。

**配置**:
```yaml
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-max
      embedding:
        options:
          model: text-embedding-v4
```

**模型清单**:

| 用途 | 模型 | 说明 |
|------|------|------|
| LLM | qwen-max | 通义千问 Max 版 |
| Embedding | text-embedding-v4 | 文本向量化 (1536维) |

### 外部搜索 (CRAG 降级)

当检索结果被判定为 `INCORRECT` 时，系统可切换到外部搜索。支持以下服务：

| 服务 | 配置 | 说明 |
|------|------|------|
| Tavily Search | `tavily.api-key=${TAVILY_API_KEY}` | AI 优化的搜索引擎 |
| Bing Search | `bing.api-key=${BING_API_KEY}` | 微软 Bing 搜索 |

如果两个 API Key 都未配置，CRAG 在 `INCORRECT` 时返回空列表。

## 监控与告警

### 访问地址

| 服务 | 地址 | 默认账号 |
|------|------|---------|
| Swagger UI | http://localhost:8080/swagger-ui.html | - |
| Actuator | http://localhost:8080/actuator | - |
| Prometheus | http://localhost:9090 | - |
| Grafana | http://localhost:3000 | admin / 123456 |
| Zipkin | http://localhost:9411 | - |
| Kibana | http://localhost:5601 | - |

### Prometheus 业务指标

| 指标名 | 类型 | 说明 |
|--------|------|------|
| `rag_cache_hit_total` | Counter | 缓存命中次数 |
| `rag_cache_miss_total` | Counter | 缓存未命中次数 |
| `rag_llm_call_total` | Counter | LLM 调用次数 |
| `rag_vector_search_total` | Counter | 向量搜索次数 |
| `rag_answer_duration_seconds` | Histogram | RAG 回答耗时 |
| `rag_llm_duration_seconds` | Histogram | LLM 调用耗时 |
| `rag_vector_search_duration_seconds` | Histogram | 向量搜索耗时 |

### 常用 PromQL

```promql
# 缓存命中率
rate(rag_cache_hit_total[5m]) / (rate(rag_cache_hit_total[5m]) + rate(rag_cache_miss_total[5m])) * 100

# P95 响应时间
histogram_quantile(0.95, rate(rag_answer_duration_seconds_bucket[5m]))

# LLM QPS
rate(rag_llm_call_total[5m])

# 活跃会话数
rag_active_sessions
```

### 告警规则

告警配置文件: `starter/src/main/resources/prometheus/rules.yml`

| 告警名称 | 条件 | 说明 |
|---------|------|------|
| ApplicationDown | `up == 0` > 1m | 应用下线 |
| HighErrorRate | `5xx 比例 > 5%` > 5m | 错误率过高 |
| HighLatency | `P95 > 5s` > 5m | 响应延迟过高 |
| HighMemoryUsage | `JVM heap > 85%` > 5m | 内存使用过高 |

钉钉通知通过 `.env` 中的 `DINGTALK_WEBHOOK_URL` 配置。

## 故障排查

### 应用启动失败

```bash
# 1. 检查 MySQL
docker-compose ps mysql
docker-compose exec mysql mysql -u root -p$DB_PASSWORD -e "SHOW DATABASES;"

# 2. 检查数据库初始化
docker-compose logs mysql | grep schema.sql

# 3. 检查应用日志
docker-compose logs -f knowledge-rag
```

常见原因:
- `.env` 中的密码与实际配置不匹配
- 数据库未初始化 (schema.sql 未执行)
- DashScope API Key 无效

### Redis 连接失败

```bash
# 检查 Redis
docker-compose ps redis
docker-compose exec redis redis-cli ping
# 应返回: PONG

# 验证密码
docker-compose exec redis redis-cli -a $REDIS_PASSWORD ping
```

### Milvus 连接失败

```bash
docker-compose ps milvus
docker-compose logs milvus | tail -50
```

常见原因:
- Milvus 启动较慢 (首次启动需初始化 ~30s)
- 向量维度不匹配 (配置为 1536，需与 Embedding 模型一致)

### Prometheus 无法抓取

```bash
# 验证指标端点
curl http://localhost:8080/actuator/prometheus

# 检查 Docker 网络
docker network inspect knowledge_monitoring
```

### 问答返回空结果

1. 检查是否已上传并向量化文档 (`POST /api/qa/ask` 返回空来源)
2. 检查 Milvus 中是否有数据
3. 检查 BM25 索引是否已填充 (`bm25Indexer.size()`)
4. 检查 CRAG 是否错误地将结果判定为 INCORRECT

## 性能调优

### JVM 参数

```bash
# 生产环境推荐
java -jar starter.jar \
  -Xms4g \
  -Xmx4g \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -XX:G1HeapRegionSize=4m \
  -XX:+HeapDumpOnOutOfMemoryError \
  -XX:HeapDumpPath=/var/log/knowledge-rag/heapdump.hprof

# GC 日志
-Xlog:gc*:file=/var/log/knowledge-rag/gc.log:time,uptime:filecount=5,filesize=10M
```

### 数据库调优

```yaml
# HikariCP 连接池
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 10
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

```ini
# MySQL my.cnf
innodb_buffer_pool_size = 2G
innodb_log_file_size = 512M
max_connections = 200
```

### Redis 调优

```yaml
spring:
  data:
    redis:
      lettuce:
        pool:
          max-active: 16
          max-idle: 8
          min-idle: 4
```

### 线程池调优

根据服务器配置和应用负载调整:

```yaml
threadpool:
  rag-retrieval:
    core-size: 10
    max-size: 20
    queue-capacity: 200
  llm-call:
    core-size: 5
    max-size: 10
    queue-capacity: 100
```

```bash
# 备份
docker-compose exec mysql mysqldump -u root -p$DB_PASSWORD knowledge_db > backup_$(date +%Y%m%d).sql
```

## 安全建议

### 上线 Checklist

- [ ] 修改所有默认密码 (MySQL root, Grafana admin)
- [ ] 生成强 JWT Secret (openssl rand -base64 32)
- [ ] 配置 HTTPS (Nginx 反向代理，Let's Encrypt)
- [ ] 限制 Actuator 端点 (prod 配置已默认隐藏 details)
- [ ] 配置防火墙 (仅开放 443 和 SSH)
- [ ] 启用日志审计
- [ ] 定期更新依赖 (mvn versions:display-dependency-updates)

### Nginx HTTPS 配置示例

```nginx
server {
    listen 443 ssl http2;
    server_name api.your-domain.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /actuator/health {
        # 允许内部监控系统访问
        allow 10.0.0.0/8;
        deny all;
        proxy_pass http://localhost:8080;
    }
}

server {
    listen 80;
    server_name api.your-domain.com;
    return 301 https://$server_name$request_uri;
}
```
