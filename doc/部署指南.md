## 快速启动

### 步骤1: 克隆项目
```bash
git clone <repository-url>
cd knowledge
```

### 步骤2: 配置环境变量
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
vim .env
```

**.env 文件内容**:
```bash
# 数据库密码
DB_PASSWORD=your_secure_password

# JWT Secret（至少32字符）
JWT_SECRET=your_256bit_secret_key_here_use_base64_encoding

# Redis 密码（可选）
REDIS_PASSWORD=

# DashScope API Key
OPENAI_API_KEY=your_dashscope_api_key

# 钉钉 Webhook（可选，用于告警通知）
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=xxx
DINGTALK_SECRET=your_secret
```

### 步骤3: 启动基础设施
```bash
# 启动所有服务（MySQL, Redis, Prometheus, Grafana, Zipkin）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 步骤4: 编译并启动应用
```bash
# 编译项目
mvn clean package -DskipTests

# 启动应用（连接 Docker 网络中的服务）
java -jar starter/target/starter-1.0-SNAPSHOT.jar \
  --spring.profiles.active=prod \
  --spring.datasource.url=jdbc:mysql://mysql:3306/knowledge_db \
  --spring.data.redis.host=redis
```

**或使用 Docker 运行应用**:
```bash
docker build -t knowledge-rag .
docker run -d \
  --name knowledge-rag \
  --network knowledge_monitoring \
  -p 8080:8080 \
  --env-file .env \
  knowledge-rag
```

### 步骤5: 验证部署
```bash
# 检查健康状态
curl http://localhost:8080/actuator/health

# 预期响应:
# {"status":"UP"}
```

## 配置说明

### 应用配置（application-prod.yml）

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://mysql:3306/knowledge_db
    username: root
    password: ${DB_PASSWORD}
    hikari:
      maximum-pool-size: 20  # 生产环境增加连接池
      minimum-idle: 10
  
  data:
    redis:
      host: redis
      port: 6379
      password: ${REDIS_PASSWORD}

  sleuth:
    sampler:
      probability: 0.1  # 生产环境降低采样率

management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus  # 隐藏敏感端点
  endpoint:
    health:
      show-details: never  # 不暴露详细信息
```

### JVM 参数优化

```bash
java -jar starter.jar \
  -Xms4g \
  -Xmx4g \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -XX:+HeapDumpOnOutOfMemoryError \
  -XX:HeapDumpPath=/var/log/knowledge-rag/heapdump.hprof
```

## 监控与告警

### 访问监控面板

| 服务 | 地址 | 账号/密码 |
|------|------|----------|
| **Grafana** | http://localhost:3000 | admin / admin |
| **Prometheus** | http://localhost:9090 | - |
| **Zipkin** | http://localhost:9411 | - |
| **Swagger UI** | http://localhost:8080/swagger-ui.html | - |
| **Actuator** | http://localhost:8080/actuator | - |

### Grafana Dashboard 配置

**推荐导入的 Dashboard**:
1. **JVM Micrometer** (ID: 4701)
   - 监控 JVM 内存、GC、线程等

2. **Spring Boot Statistics** (ID: 11378)
   - 监控 HTTP 请求、延迟、错误率

3. **RAG 自定义 Dashboard**（手动创建）
   - 缓存命中率
   - LLM 调用次数
   - RAG 响应时间分布

**导入步骤**:
1. 登录 Grafana
2. Create → Import
3. 输入 Dashboard ID
4. 选择 Prometheus 数据源
5. 点击 Import

### 告警配置

**告警规则**: 见 `starter/src/main/resources/prometheus/rules.yml`

**告警通知**: 
- 默认通过钉钉 Webhook 发送
- 配置 `.env` 中的 `DINGTALK_WEBHOOK_URL`

**测试告警**:
```bash
# 停止应用触发 ApplicationDown 告警
docker stop knowledge-rag

# 等待 1 分钟
# 检查钉钉是否收到告警消息
```

## 故障排查

### 问题1: 应用启动失败

**症状**: `docker-compose logs` 显示连接拒绝

**排查步骤**:
```bash
# 1. 检查 MySQL 是否启动
docker-compose ps mysql

# 2. 检查数据库连接
docker-compose exec mysql mysql -u root -p$DB_PASSWORD -e "SHOW DATABASES;"

# 3. 查看应用日志
docker-compose logs -f knowledge-rag
```

**解决方案**:
- 确保 `.env` 中的 `DB_PASSWORD` 与 MySQL 一致
- 检查 `schema.sql` 是否正确执行

### 问题2: Redis 连接失败

**症状**: 日志显示 `Cannot get Jedis connection`

**排查**:
```bash
# 检查 Redis 状态
docker-compose ps redis

# 测试 Redis 连接
docker-compose exec redis redis-cli ping
# 应返回: PONG
```

**解决方案**:
- 检查 `REDIS_PASSWORD` 配置
- 重启 Redis: `docker-compose restart redis`

### 问题3: Prometheus 无法抓取指标

**症状**: Prometheus Targets 页面显示 DOWN

**排查**:
```bash
# 检查 Actuator 端点
curl http://localhost:8080/actuator/prometheus

# 检查 Docker 网络
docker network inspect knowledge_monitoring
```

**解决方案**:
- 确保应用在 Docker 网络中可访问
- 修改 `prometheus.yml` 中的 target 为容器名

### 问题4: Grafana 无数据

**症状**: Dashboard 显示 "No data"

**排查**:
1. 检查 Prometheus 数据源配置
2. 确认 Prometheus 正在抓取指标
3. 在 Prometheus 中执行查询测试

**解决方案**:
```promql
# 在 Prometheus 中测试
up{application="knowledge-rag"}

# 应返回: 1
```

## 性能调优

### JVM 调优

**G1 GC 参数**:
```bash
-Xms4g -Xmx4g
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
-XX:G1HeapRegionSize=4m
-XX:+ParallelRefProcEnabled
```

**监控 GC**:
```bash
# 启用 GC 日志
-Xlog:gc*:file=/var/log/knowledge-rag/gc.log:time,uptime:filecount=5,filesize=10M
```

### 数据库调优

**HikariCP 连接池**:
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 10
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

**MySQL 优化**:
```ini
# my.cnf
innodb_buffer_pool_size = 2G
innodb_log_file_size = 512M
max_connections = 200
```

### Redis 调优

```yaml
spring:
  data:
    redis:
      lettuce:
        pool:
          max-active: 16
          max-idle: 8
          min-idle: 4
```

### 线程池调优

```yaml
# application.yml
threadpool:
  rag-retrieval:
    core-size: 10
    max-size: 20
    queue-capacity: 200
  llm-call:
    core-size: 5
    max-size: 10
    queue-capacity: 100
```

## 备份与恢复

### 数据库备份

```bash
# 备份
docker-compose exec mysql mysqldump -u root -p$DB_PASSWORD knowledge_db > backup.sql

# 恢复
docker-compose exec -T mysql mysql -u root -p$DB_PASSWORD knowledge_db < backup.sql
```

```bash
# 启用 RDB 持久化
docker-compose exec redis redis-cli BGSAVE

# 备份 dump.rdb
docker cp knowledge-redis:/data/dump.rdb ./backup/
```

### Nginx HTTPS 配置示例

```nginx
server {
    listen 443 ssl;
    server_name api.example.com;

    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
